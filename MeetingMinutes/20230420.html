Discussed Points
Digital Pipeline
There should ba a Digtial Pipeline to align with MLB to create resource plan, and then allocate resource and prioritize resource. (e.g. Some of the Digital Projects may need to integrate with Salesforce. Thus, a Digital Pipeline booking Salesforce resource is critical.).
Digital Project Management Methodology
There are various types of Digital Projects. Which Project Management Methodology to choose? It depends on various aspects. Since then, there should be a Project Management Methodology decision mechanism.
An interim solution for Digital Project Management Methodology Selection
As per there is no Digital Project Management Methodology Decision Mechanism in place at the moment. An interim solution is: before the project kick off, align with project team for which Project Management Methodology will be used for this project.
Define Procurement Role for Digital Projects
A Digital Procurement Lead will need to have both hardware and software knowledge so that he/she could make decision during the vendor evalustion and selection process. IF there is already a role in place, THEN he/she could be involed in digital project procuremnt process. ELSE digtial procurment capability will need to be setup.
Vendor Evaluation Matrix
There is a Vendor Evaluation Matrix in IRPDP Templates. It could be a reference to create a Digital Vendor Evaluation Matrix.
IT Support for Digital Projects
IT support will be needed from the following aspects in a Digital Project:
  • (1) Infra
  • (2) Security
  • (3) System Integaraion
  • (4) User Account Management
  • (5) System Landscape
DevOps Resource
DevOps resource under Engineering.
Agile Roles need to be filled in.
As per the coming needs of Digital Growth, the current Digtial Organization may need to have more Agile Roles to be filled in.
Digital Project Gate Review Stakeholders
Currently, Digital Project Gate Review Stakeholders are Flora and Lily. It depends on projects. More stakeholders can be added later.
